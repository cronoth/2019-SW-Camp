SW Camp 개발 과제(2019.10.08)

한국무역정보통신 2019년 신입직원 SW Camp 1일 차 개발 과제 안내문

주제 안내

문서 난독화 변환

  • 허용된 문자셋 안에서 같은 종류의 문자로 치환하여 해당 내용을 난독화

  • 구현하고자 하는 변환 규칙을 서술하고, 해당 내용을 프로그래밍(혹은 기술)

  • 유추하기 어려운 난독화 알고리즘을 착안하여 오류없이 동작하도록 작성

지시사항

  • 기본조건

    입출력 문서 모두 EUC-KR 인코딩 사용
    입출력 문서의 크기 변경은 없어야 함

    개행문자 및 공백문자는 원본 문서대로 유지
    쉽게 원본 문자열을 유추할 수 없어야 함

  • 허용범위

    숫자 / 영어 / 한글 / (일부) 특수문자 사용 가능
    원본과 변환본은 동일한 문자셋으로만 치환 처리

    예시 : 123 -> 321, abc -> ABC, ㅂㅈㄷ -> ㄸㄲㅆ, !@# -> $%^

  • 특수문자 처리

    특수문자의 경우 키보드로 입력가능한 문자에 한함
    ※ 제어문자 및 전각문자 등 금지

    일부 특수문자의 경우 Escape Character를 유의하여 작성

SW Camp 과제 기술서 가이드

한국무역정보통신 2019년 신입직원 SW Camp 과제 기술서 작성안내

개요

  • 각 프로그래밍 과제에 대한 기술서 작성 및 제출

  • 작성한 과제 기술서를 바탕으로 개인별 평가 시행

작성방법

  • stackedit.io 마크다운 문서편집 페이지 활용

    초기 튜토리얼 페이지 - 좌측 [편집화면] | 우측 [결과화면]
    상단바 좌측 [폴더] 모양 버튼 - FILE 편집 가능

  • 본 안내문과 같이 마크다운 문서 형식으로 작성하여 PDF로 출력

    마크다운 사용법 (https://vo.la/7Xiv) 참조
    가급적 크롬 브라우저의 ‘PDF로 저장’ 프린터 사용

  • 메뉴에서 구글 계정과 연동하여 작성한 문서를 보관처리 가능

    상단바 우측 [#] 모양 버튼 - MENU 선택 가능
    Templates, Import/Export 등 Workspace 관리

작성양식

  • 자유양식 (필수 구성요소 포함하여 작성)

  • 프로그램 사양서(필수)

    구현 언어, I/O 지원방식, 사용 방법 등

    * Web Based 전자문서 변환 프로그램
 . 언어 및 도구 : Java/Spring Framework + Tomcat
 . I/O 지원방식 : HTML Form Key-In 및 출력 XML 파일 다운로드
 . 사용 방법 : 로컬서버를 구동하여 변환 대상 문서 및 메타데이터 입력
 . 구현 의도 : 웹페이지에서 쉽고 빠르게 작성하여 즉시 결과 확인

  • 구현 알고리즘 기술(필수)

    특징, 장점, 성능 등 개발의도 표현

    * Java Object XML Mashalling
 . 특징 : JAXB Annotation으로 직관적이고 쉽게 맵핑 처리
 . 장점 : 선택 객체만 맵핑 가능(상태관리 등 private 객체 숨김)
 . 동작설명 : 페이지에서 입력하고자 하는 Element field를 추가하여,
             name/value를 설정하고 서버에서 XML Mashalling 처리
 . 예외처리 : root Element 입력은 필수이며, 예외 발생시 child
 	         Element(name:errorInfomation)에서 해당 메시지 출력

  • 핵심기능 및 주석 설명(선택)

    개인별 발표 시간 내에 설명 가능한 분량으로 중요 부분만 요약

    return 9 - src;	// 0 ~ 9 사이의 숫자를 입력받아 변환한다.
